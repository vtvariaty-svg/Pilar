firebase.json:
- Configurar hosting para publicar a pasta dist.
- Configurar rewrite de SPA:
  source: "**"
  destination: "/index.html"
- Incluir Firestore rules.
- Incluir Storage rules.
- Incluir indexes se necessário.

Firestore Security Rules:
Criar regras seguras.

Requisitos:
- Qualquer visitante pode criar lead na collection leads.
- Visitante não pode ler leads.
- Visitante não pode atualizar leads.
- Visitante não pode deletar leads.
- Usuário autenticado pode ler leads.
- Usuário autenticado pode atualizar status e internalNotes.
- Usuário autenticado pode deletar apenas se estiver autenticado.
- Validar campos obrigatórios.
- Validar tipos básicos.
- Validar status permitido.
- Validar source como "site".
- Não deixar Firestore aberto publicamente.

Storage Rules:
- Permitir leitura pública de imagens do portfólio.
- Permitir upload, update e delete apenas para usuário autenticado.
- Validar que arquivos enviados sejam imagens.
- Validar limite de tamanho razoável, por exemplo até 5 MB.

Observação importante:
Firebase Security Rules precisam ser tratadas como parte do produto, não como detalhe. Não deixe regras abertas do tipo allow read, write: if true.

Deploy obrigatório via Firebase CLI:
Depois de implementar, configure o projeto para deploy com Firebase CLI.

Preciso que você:
1. Verifique se firebase-tools está disponível.
2. Se não estiver, me diga para instalar com:
   npm install -g firebase-tools
3. Prepare o projeto com os arquivos necessários do Firebase.
4. Garanta que npm run build gera a pasta dist.
5. Garanta que firebase.json aponta para dist.
6. Garanta que o rewrite de SPA está configurado.
7. Me oriente a executar:
   firebase login
   firebase use --add
   firebase deploy
8. Se possível no ambiente atual, execute o build local.
9. Se o ambiente já estiver autenticado no Firebase CLI e tiver projeto configurado, execute o deploy.
10. Se não puder executar o deploy por falta de autenticação, deixe tudo pronto e me diga exatamente quais comandos executar.
